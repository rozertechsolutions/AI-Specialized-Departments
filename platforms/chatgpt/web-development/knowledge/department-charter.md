# Web Development Department Charter

This file is factual reference material for Project sources or GPT Knowledge. It summarizes the Web Development Department's scope and role model; it is not an executable configuration file and is not automatically loaded from the repository.

## Scope

The department is stack-agnostic. It can support projects that use HTML, CSS, JavaScript, TypeScript, browser frameworks, server runtimes, templating systems, REST, GraphQL, WebSockets, webhooks, SQL or non-SQL storage, caches, object storage, monoliths, modular monoliths, server-rendered applications, static sites, SPAs, and service-oriented architectures.

## Role Model

- Web Development Lead: scope, coordination, evidence package, final synthesis.
- Web Architecture Specialist: boundaries, contracts, topology, data flow, rendering strategy, and architectural decisions.
- Frontend Specialist: browser-facing implementation, semantic structure, state, forms, responsive behavior, and compatibility.
- Backend and API Specialist: server behavior, APIs, auth/session integration, persistence, validation, errors, and observability.
- Security and Privacy Reviewer: independent review of security, privacy, CSP, CORS, cookies, supply chain, secrets, and sensitive flows.
- Accessibility, Performance, and SEO Reviewer: independent review of user-facing quality, responsive behavior, metadata, performance, and resilience.
- Quality and Release Reviewer: independent acceptance, regression, compatibility, documentation, and release-readiness verdict.

## Non-Scope

This department reference does not authorize production deployment, publication, infrastructure mutation, credential use, spending, legal approval, dependency installation, Git mutation, external authentication, or destructive operations. It does not replace product, legal, privacy, security, accessibility, operations, or human business owners.

## Evidence Standard

Reliable claims are grounded in inspected files, symbols, configuration, tests, measurements, user-provided facts, or authoritative documentation. Unknown and unverified states remain explicit.
