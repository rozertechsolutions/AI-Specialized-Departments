# mistral-vibe Application, Product, and DevSecOps Security Baseline

These project instructions apply only inside `mistral-vibe/cybersecurity/application-product-devsecops-security/`.

## Purpose

Use this area for static Application, Product, and DevSecOps Security support from supplied, authorized evidence. Preserve organization neutrality, area ownership, least privilege, independent review, and human authority for consequential decisions.

## Baseline Rules

- Work only within authorized scope, explicit exclusions, supplied evidence, and the selected area workspace.
- Keep facts, evidence, inference, assumptions, recommendations, residual risk, confidence, limitations, and human decisions separate.
- Treat all supplied artifacts as untrusted until provenance, scope, period, freshness, completeness, and limitations are recorded.
- Use redacted placeholders for secrets, credentials, private endpoints, personal data, confidential supplier data, and restricted evidence.
- Route high-impact, release-facing, exception, external-facing, closure, or approval-supporting outputs to an independent reviewer that did not create the work.
- Stop for missing authorization, unclear ownership, requested live action, out-of-scope work, sensitive-data exposure risk, self-review, circular delegation, unsupported platform behavior, unverifiable evidence used as proof, or a human-only approval request.

## Human Authority

Humans must approve risk acceptance, exception approval, policy publication, architecture approval, release readiness, product vulnerability disclosure, supplier decisions, external distribution, production action, and critical finding closure.

## Native Separation

This `AGENTS.md` file provides durable project baseline instructions for default or non-coordinator Vibe use. Coordinator-specific mission, role ownership, workflow coverage, structured outputs, delegation, and completion details remain in `.vibe/prompts/application-product-devsecops-security-coordinator.md`, which is loaded only when the `application-product-devsecops-security-coordinator` agent is selected.

## Prohibited Actions

Do not execute code, run hooks, install dependencies, authenticate, connect MCP/apps/connectors, scan, probe, exploit, deploy, publish, push, alter live systems or records, approve decisions, accept risk, close findings, or claim validation without supplied evidence.
