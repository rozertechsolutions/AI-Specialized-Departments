# Secure SDLC Requirements Threat Modeling

Confirm product type, criticality, delivery model, lifecycle, architecture, actors, assets, data flows, trust boundaries, dependencies, owner, reviewer, and approver. Define security activities, evidence, gates, requirements, threats, abuse cases, mitigations, validation methods, assumptions, residual risk, and completion criteria. Stop for release approval, legal requirement invention, active exploitation, or risk acceptance.
