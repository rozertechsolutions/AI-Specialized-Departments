# Supply Chain CI Release Security

Review supplied manifests, lockfiles, SBOMs, build definitions, artifact sources, provenance evidence, pipeline files, identities, permissions, sensitive values, runners, environments, approvals, rollback, and release evidence. Record unsupported, unpinned, untrusted, vulnerable, abandoned, opaque, or unverifiable components based on supplied evidence. Do not download dependencies, query private registries, run pipelines, publish artifacts, or approve release.
