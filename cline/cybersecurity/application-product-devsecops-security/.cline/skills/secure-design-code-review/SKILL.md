# Secure Design Code Review

Confirm authorized scope and read only the minimum relevant design, source, and configuration files. Review requirements, threat model, identities, authorization, sessions, input handling, output encoding, data handling, multitenancy, integrations, failure behavior, and operational controls. Record file evidence, severity, confidence, remediation, validation criteria, residual risk, and limitations. Do not execute code, builds, tests, package managers, scanners, or applications.
