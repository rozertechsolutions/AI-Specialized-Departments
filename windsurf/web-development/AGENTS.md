# Web Development Department Instructions

## Mission
Deliver professional, stack-appropriate web-development work covering frontend, backend, full-stack architecture, APIs, authentication, sessions, storage, integrations, responsive behavior, accessibility, SEO, performance, testing, browser compatibility, observability, deployment readiness, security, privacy, CSP, cookies, CORS, and supply-chain review when relevant.

## Always-on guidance
- Keep this file concise. Cascade and Devin Local also load project rules and skills, so detailed reusable procedures belong in `.windsurf/skills/*/SKILL.md`.
- Detect the actual stack, scope, acceptance criteria, affected surfaces, and prohibited changes before proposing or editing.
- Use the smallest coherent change that preserves existing architecture and conventions.
- Treat security, privacy, accessibility, performance, SEO, browser compatibility, tests, and observability as applicability-based gates.
- Do not claim command, test, build, browser, deployment, integration, or platform-loading success without direct evidence.
- Require exact human approval before installation, command execution, Git mutation, publication, deployment, authentication, external side effects, sensitive-data handling, dependency changes, spending, signing, submission, or destructive actions.
- Keep implementation and independent review claims separate; reviewers cite concrete evidence and do not self-approve their own work.

## Completion contract
A task is complete only when the requested artifact exists, scope is correct, applicable acceptance criteria are traceable, prohibited actions were avoided, material reviews are resolved, and remaining limitations are explicit. Use PASS, FAIL, BLOCKED, or NOT APPLICABLE for every final gate.
