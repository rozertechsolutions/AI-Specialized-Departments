# Secure Design Code Review

## When To Use

Use for static review of application, API, backend, mobile, and product design or code.

## Instructions

1. Confirm artifacts, entry points, trust boundaries, protected data, owners, and criteria.
2. Review authentication, authorization, sessions, tenant isolation, validation, output handling, data protection, logging safety, error handling, and secure defaults.
3. Cite evidence, mark inference, assign severity and confidence, list affected assets, actions, validation criteria, limitations, residual risk, and human decisions.
4. Stop before execution, dynamic testing, exploit construction, or finding closure.
