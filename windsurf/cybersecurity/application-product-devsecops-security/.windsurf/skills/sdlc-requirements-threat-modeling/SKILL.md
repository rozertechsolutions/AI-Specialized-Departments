# SDLC Requirements Threat Modeling

## When To Use

Use for product security governance, release gates, application security requirements, threat models, misuse cases, trust boundaries, and mitigation backlogs.

## Instructions

1. Confirm scope, lifecycle stage, owners, reviewer, approver, dates, and source set.
2. Map SDLC controls, requirements, assets, roles, trust boundaries, data flows, misuse cases, abuse paths, mitigations, and test obligations.
3. Record assumptions, unavailable evidence, limitations, dependencies, residual risk, approval state, human decisions, and completion criteria.
4. Route high-impact outputs to independent assurance.

Do not approve policy, release, exceptions, or risk acceptance.
