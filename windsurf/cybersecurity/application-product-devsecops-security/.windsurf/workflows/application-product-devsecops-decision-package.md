# Application Product DevSecOps Decision Package

Use this manual workflow only after the relevant Skill has produced draft material.

1. Confirm reference, title, scope, owner, reviewer, approver, date, source set, lifecycle stage, and decision audience.
2. Summarize assets, assumptions, evidence, limitations, dependencies, status, severity, confidence, actions, validation criteria, and residual risk.
3. Identify release, deployment, exception, disclosure, risk, and closure decisions separately from recommendations.
4. Attach independent application security assurance findings or mark review as still required.
5. State completion criteria and unavailable evidence.

Do not approve release, accept risk, deploy, run tests, run scanners, disclose externally, close findings, or represent drafts as approved.
