# Identity Cloud Network Data Design

Use this procedure for target-state security architecture across identity, cloud, network, communications, data protection, cryptography, keys, and restricted material.

1. Confirm environment scope, administration paths, cloud boundaries, network paths, data classes, and cryptography requirements.
2. Map control patterns for access, segmentation, communications, data protection, key handling, and restricted material.
3. Record assumptions, evidence, dependencies, residual risk, human decisions, and approval needs.
4. Return a target-state design and review notes with status, severity, confidence, limitations, and actions.
5. Request `independent-architecture-reviewer` before completion.

Stop for access changes, live configuration, key operations, monitoring operations, or sensitive material exposure.
