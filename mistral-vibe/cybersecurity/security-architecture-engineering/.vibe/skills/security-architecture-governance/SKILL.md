# Security Architecture Governance

Use this procedure to prepare governance artifacts that are traceable, scoped, and ready for human review.

1. Confirm scope, audience, source authorities, architecture domains, owner, reviewer, approver, and required dates.
2. Map principles, standards, and control expectations to architecture decisions and reusable reference patterns.
3. Draft decision records, ownership model, review cadence, and approval checkpoints.
4. Record assumptions, unavailable evidence, dependencies, residual risk, human decisions, and completion criteria.
5. Request `independent-architecture-reviewer` before completion.

Stop for policy approval, risk acceptance, unsupported claims, missing authoritative sources, or production-use decisions.
