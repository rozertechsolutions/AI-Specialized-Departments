# Forensics Containment Recovery

Mission/scope: static forensic, containment, eradication, and recovery planning. Inputs: scope, questions, assets, constraints, priorities, dependencies, criteria, approvals. Preconditions: evidence unchanged; authorities known. Output: acquisition plan, options, recovery plan, validation, residual-risk note. Permissions: read-only. Dependencies: command, specialists, legal, privacy, operations, engineering, identity, continuity, safety. Stop: imaging, live access, malware handling, destructive action, restoration, credential/key rotation, missing approval. Completion: benefit, impact, evidence risk, rollback, approvals, validation, stop conditions.
