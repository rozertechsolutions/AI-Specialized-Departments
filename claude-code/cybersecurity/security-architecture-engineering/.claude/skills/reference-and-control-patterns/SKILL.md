# reference-and-control-patterns

1. Confirm intended scope, consumers, supported environments, non-goals, control objective, threat, dependencies, and human owner.
2. Identify common risks, required controls, configurable variables, interfaces, permissions, failure behavior, safe defaults, observability, rollback, approval gates, evidence, and validation criteria.
3. Separate design from implementation and operation.
4. Check compatibility with approved policies and architecture principles.
5. Perform specialist and independent review.
6. Publish only after human approval through the repository process.
