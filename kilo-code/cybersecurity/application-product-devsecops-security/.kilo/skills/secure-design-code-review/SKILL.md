# Secure Design Code Review

1. Confirm supplied artifacts, in-scope components, protected data, trust boundaries, entry points, owners, and review criteria.
2. Review authentication, authorization, session handling, tenant isolation, validation, output handling, data protection, logging, error handling, and secure defaults.
3. Cite exact evidence and mark inference separately.
4. Provide findings with severity, confidence, affected assets, remediation options, validation criteria, limitations, residual risk, and human decisions.
5. Stop before code execution, dynamic testing, exploit construction, or release approval.
