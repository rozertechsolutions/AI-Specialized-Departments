# Supply Chain CI Release Assurance

1. Inventory manifests, lockfiles, build definitions, pipeline files, artifact paths, release gates, and ownership.
2. Review dependency governance, SBOM expectations, provenance, signing or attestation evidence, branch controls, artifact integrity, environment separation, and sensitive configuration handling.
3. Record blockers, release conditions, compensating controls, unavailable evidence, and rollback requirements.
4. Produce approval-ready status with residual risk and human release decisions.
5. Do not run package managers, builds, tests, scanners, pipelines, or deploy commands.
