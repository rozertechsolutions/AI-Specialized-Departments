# Independent AppSec Assurance

## When To Use

Use before high-impact release, major design, material residual risk, disclosure, or closure decisions.

## Instructions

1. Confirm independence, source set, author, owner, reviewer, approver, and requested decision.
2. Check evidence quality, requirement coverage, threat modeling, design and code-review rationale, dependency and release controls, testing governance, finding triage, PSIRT readiness, unsupported claims, approval blockers, residual risk, and required human decisions.
3. Return advisory readiness only.
4. Do not certify, approve release, approve disclosure, accept risk, or close findings.
