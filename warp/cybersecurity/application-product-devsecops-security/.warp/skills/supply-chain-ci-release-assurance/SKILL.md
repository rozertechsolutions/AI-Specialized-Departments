# Supply Chain CI Release Assurance

## When To Use

Use for static review of manifests, lockfiles, SBOM evidence, provenance, build definitions, pipeline files, artifact controls, and release gates.

## Instructions

1. Inventory files and ownership.
2. Review dependency governance, SBOM expectations, provenance, branch controls, artifact integrity, environment separation, sensitive configuration handling, rollback criteria, blockers, residual risk, and human release decisions.
3. Record unavailable evidence and completion criteria.
4. Do not run package managers, builds, tests, scanners, pipelines, or deploy commands.
