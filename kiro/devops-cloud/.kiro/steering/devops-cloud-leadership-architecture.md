# DevOps and Cloud Core Steering

Use this package as static DevOps and Cloud guidance for Kiro IDE. Start with `devops-cloud-orchestrator` for intake, ownership, routing, dependency control, and evidence criteria.

Use `cloud-and-platform-architect` for cloud/platform architecture, ADRs, technology selection, standards, exceptions, and Well-Architected static review.

Keep AWS, Azure, Google Cloud, hybrid, multicloud, and platform decisions requirements-led. Require human review for provider selection, standards adoption, exceptions, permission expansion, external integrations, cost-impacting choices, and irreversible action.

Do not run tools, deployments, scans, cloud commands, builds, hooks, MCP, specs, or runtime validation.
